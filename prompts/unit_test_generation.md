# 목적

아래 개발 의뢰와 소스 변경 근거를 바탕으로 단위테스트 문안을 작성한다.

# 작성 규칙

- 테스트 절차는 정확히 3개이며 각 단계는 한 줄로 작성한다.
- 테스트 사전조건은 정확히 3개이며 각 항목은 한 줄로 작성한다.
- 화면 배치Step 보고서 ID와 이름은 근거에서 확인되는 값만 사용한다.
- 테스트 데이터와 예상결과는 각각 한 문장으로 간결하게 작성한다.
- 처리내역은 변경 주제별 제목과 한 줄 설명으로 작성한다.
- 테스트내역은 정확히 3개의 확인 문장으로 작성한다.
- 결과 확인내역은 핵심 검증 항목을 1개 이상 5개 이하로 작성한다.
- 프로그램 파일명과 신규 변경 삭제 구분은 응답에 만들지 않는다.
