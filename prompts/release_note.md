# 작성 목표

팀원에게 공유할 릴리즈 안내 메일을 작성한다.

- subject는 변경 대상과 확인 요청이 드러나는 자연스러운 메일 제목으로 작성한다.
- body는 반드시 안녕하세요로 시작하고 감사합니다로 끝낸다.
- 첫 문단에서 이번 변경의 목적을 짧게 설명한다.
- 주요 변경 사항은 실제 변경 주제 중심으로 최대 다섯 개까지 정리한다.
- 확인 부탁드리는 내용은 최종 단위테스트 문안에서 실제로 검증할 수 있는 내용만 안내한다.
- 단순한 변경은 짧게 쓰고 파일 수가 많다는 이유로 내용을 늘리지 않는다.
- AI가 작성한 보고서처럼 설명하지 말고 같은 팀 동료에게 보내는 업무 메일 문체를 사용한다.
